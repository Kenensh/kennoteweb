介紹

Introduction

React Native 是一個由 Facebook 開發的開源框架，它允許開發者使用 JavaScript 和 React 來構建原生移動應用。 與傳統的混合應用不同，React Native 應用使用與原生應用相同的 UI 組件，從而提供更好的性能和用戶體驗。

React Native is an open-source framework developed by Facebook that allows developers to use JavaScript and React to build native mobile applications. Unlike traditional hybrid apps, React Native apps use the same UI components as native apps, providing better performance and user experience.

React Native 的口號是「一次學習，到處編寫」，強調跨平台開發的能力。 React Native's tagline is "Learn once, write anywhere," emphasizing its cross-platform development capabilities.

環境設置

Environment Setup

開始使用 React Native 開發前，您需要設置開發環境。有兩種主要的方法可以創建 React Native 應用：

Before starting development with React Native, you need to set up your development environment. There are two main ways to create React Native applications:

使用 Expo CLI

Using Expo CLI

Expo 是一個圍繞 React Native 構建的工具集，它簡化了開發過程：

Expo is a set of tools built around React Native that simplifies the development process: 

# 安裝 Expo CLI
npm install -g expo-cli

# 創建新項目
expo init MyProject

# 啟動開發服務器
cd MyProject
expo start

使用 React Native CLI

Using React Native CLI

對於需要更多定制化和原生模塊的項目，可以使用 React Native CLI：

For projects that require more customization and native modules, you can use the React Native CLI: 

# 安裝 React Native CLI
npm install -g react-native-cli

# 創建新項目
npx react-native init MyProject

# 啟動開發服務器
cd MyProject
npx react-native start

# 在 Android 上運行
npx react-native run-android

# 在 iOS 上運行
npx react-native run-ios

基本組件

Basic Components

React Native 提供了一系列基本組件，它們對應於各平台上的原生 UI 元素：

React Native provides a series of basic components that correspond to native UI elements on each platform:

  • View - 容器組件，相當於網頁中的 div
  • Text - 顯示文本
  • Image - 顯示圖片
  • ScrollView - 可滾動的容器
  • FlatList - 高效顯示長列表數據
  • TextInput - 文本輸入框
  • Button - 按鈕
  • TouchableOpacity - 可點擊的組件，有按下效果
  • Switch - 開關組件

基本應用示例

Basic Application Example

import React, { useState } from 'react';
import { View, Text, TextInput, Button, StyleSheet } from 'react-native';

const App = () => {
  const [name, setName] = useState('');
  const [greeting, setGreeting] = useState('');

  const handleGreet = () => {
    setGreeting(`Hello, ${name}!`);
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Welcome</Text>
      <TextInput
        style={styles.input}
        placeholder="Enter your name"
        value={name}
        onChangeText={setName}
      />
      <Button title="Greet" onPress={handleGreet} />
      {greeting ? <Text style={styles.greeting}>{greeting}</Text> : null}
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    alignItems: 'center',
    justifyContent: 'center',
    padding: 20,
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 20,
  },
  input: {
    width: '100%',
    height: 40,
    borderWidth: 1,
    borderColor: '#ccc',
    borderRadius: 5,
    marginBottom: 20,
    paddingHorizontal: 10,
  },
  greeting: {
    marginTop: 20,
    fontSize: 18,
  },
});

export default App;

樣式與佈局

Styling and Layout

React Native 中的樣式採用類似 CSS 的語法，但使用 JavaScript 對象。佈局則主要基於 Flexbox：

Styling in React Native uses a syntax similar to CSS but with JavaScript objects. Layouts are primarily based on Flexbox:

StyleSheet API

StyleSheet API

import { StyleSheet } from 'react-native';

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'center',
  },
  text: {
    fontSize: 16,
    color: '#333',
    fontWeight: 'bold',
  },
});

Flexbox 佈局

Flexbox Layout

React Native 使用 Flexbox 進行佈局，主要屬性包括：

React Native uses Flexbox for layout, with key properties including:

  • flex - 定義元素如何填充可用空間
  • flexDirection - 主軸方向（'row', 'column'）
  • justifyContent - 主軸對齊方式
  • alignItems - 交叉軸對齊方式
  • flexWrap - 是否允許換行
React Native 中的 Flexbox 與網頁有些微差異，如默認的 flexDirection 是 'column' 而非 'row'。 Flexbox in React Native has some subtle differences from the web, such as the default flexDirection being 'column' instead of 'row'.

導航

Navigation

React Native 應用中的導航通常使用 React Navigation 庫實現：

Navigation in React Native applications is typically implemented using the React Navigation library: 

# 安裝 React Navigation
npm install @react-navigation/native

# 安裝依賴
npm install react-native-screens react-native-safe-area-context

# 安裝導航類型（例如堆棧導航）
npm install @react-navigation/stack

基本導航設置

Basic Navigation Setup

import React from 'react';
import { NavigationContainer } from '@react-navigation/native';
import { createStackNavigator } from '@react-navigation/stack';
import HomeScreen from './screens/HomeScreen';
import DetailsScreen from './screens/DetailsScreen';

const Stack = createStackNavigator();

function App() {
  return (
    <NavigationContainer>
      <Stack.Navigator initialRouteName="Home">
        <Stack.Screen
          name="Home"
          component={HomeScreen}
          options={{ title: 'Overview' }}
        />
        <Stack.Screen name="Details" component={DetailsScreen} />
      </Stack.Navigator>
    </NavigationContainer>
  );
}

export default App;

導航類型

Navigation Types

  • 堆棧導航 (Stack) - 頁面疊加式導航
  • Stack Navigation - Pages stack on top of each other
  • 標籤導航 (Tab) - 底部或頂部選項卡導航
  • Tab Navigation - Bottom or top tabs navigation
  • 抽屜導航 (Drawer) - 側邊欄菜單導航
  • Drawer Navigation - Side menu navigation

持久化存儲

Persistent Storage

對於需要保存在設備上的數據，可以使用 AsyncStorage 或更現代的替代方案：

For data that needs to be saved on the device, you can use AsyncStorage or more modern alternatives: 

import AsyncStorage from '@react-native-async-storage/async-storage';

// 存儲數據
const storeData = async (key, value) => {
  try {
    await AsyncStorage.setItem(key, JSON.stringify(value));
  } catch (e) {
    console.error('Error saving data', e);
  }
};

// 讀取數據
const getData = async (key) => {
  try {
    const value = await AsyncStorage.getItem(key);
    return value != null ? JSON.parse(value) : null;
  } catch (e) {
    console.error('Error reading data', e);
    return null;
  }
};

性能優化

Performance Optimization

優化 React Native 應用的性能對於提供良好的用戶體驗至關重要：

Optimizing the performance of React Native applications is crucial for providing a good user experience:

常見優化技巧

Common Optimization Techniques

  • 使用 FlatList 代替 ScrollView 處理長列表
  • Use FlatList instead of ScrollView for long lists
  • 使用 memo 和 useCallback 減少不必要的渲染
  • Use memo and useCallback to reduce unnecessary rendering
  • 避免匿名函數 作為 props 傳遞
  • Avoid anonymous functions as props
  • 使用圖片緩存 提高圖片加載性能
  • Use image caching to improve image loading performance
  • 使用 Hermes 引擎 提高 JavaScript 執行速度
  • Use the Hermes engine to improve JavaScript execution speed

性能監控

Performance Monitoring

  • 使用 React Native 開發者菜單 檢查性能指標
  • Use the React Native Developer Menu to check performance metrics
  • 使用 Flipper 進行高級調試和性能分析
  • Use Flipper for advanced debugging and performance analysis

  • Flipper - Facebook 的移動應用調試平台
  • Flipper - Facebook's mobile app debugging platform

常用調試技巧

Common Debugging Techniques
